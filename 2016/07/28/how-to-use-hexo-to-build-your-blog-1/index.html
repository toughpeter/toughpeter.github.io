使用 Hexo 與 Github 架設自己的 Blog

2016-07-28
Programming

Hexo 是什麼？

根據 Hexo的官方網站 的介紹， Hexo 是一個快速、簡單且強大的網誌框架，讓你可以輕易的管理、發佈文章。
如果你不熟悉 Coding 這件事，說簡單其實也沒那麼簡單，當初選擇使用 Hexo 的原因是因為 Hexo 可以使用 Markdown 的格式撰寫文章，並在幾秒鐘內部署到 Github 上，Hexo 也有許多漂亮的主題可以直接使用，當然如果你想自己刻一套主題也是沒問題的。

我選擇使用 Hexo 的原因

  1. 客製化方便
  2. 搭配 Github 使用，方便快速
  3. 純粹為了學新東西

如何利用 Hexo 部署到 Github 上產生靜態網頁？

在我們開始之前你要先有

  1. 安裝 Hexo
  2. 在 Github上開一個命名為自己Github帳號.github.io的 Repository，例如：「username.github.io」
  3. 產生 Hexo blog 的靜態檔案
  4. 上傳到 Github 上

安裝 Hexo

透過 npm 安裝 Hexo

如果你還沒用過 node.js 與 npm，可以參考 Hexo官網的安裝流程。

如果你已經熟悉npm我們就來直接安裝吧！(如果是Mac可能會需要在指令前面加上sudo)

測試是否成功安裝 Hexo

新增一個 Hexo 專案

在目前的資料夾中產生一個叫做「Blog」的 Hexo 專案，並在終端機切到該資料夾，再透過 npm 安裝相關套件。

Hexo 會自動新增一篇範例文章，在本地端把 Server 跑起來，預設可以在 http://localhost:4000/ 看到內容。

常用指令介紹

新增一篇文章

在 source/_post 中建立一個叫做「new_post_name_goes_here.」的新文章，在 source/_post 經過 generate 之後會產生靜態檔案。

新增一篇草稿

在source/_draft中建立草稿文章。

發表草稿

在source/_draft裡面找到指定的草稿並移到source/_post中。
