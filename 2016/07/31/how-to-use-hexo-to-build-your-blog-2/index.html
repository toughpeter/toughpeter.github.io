修改 Hexo 樣式主題並設定 Url 與 Sitemap

2016-07-31
hexo

上一篇介紹如何使用 Hexo 與 Github 架設自己的 Blog，這篇會記錄：

  1. 如何把主題樣式修改成其他版型
  2. 如何設定成自己想要的網址
  3. 如何上傳 Sitemap

設定 Hexo 的主題（Theme）

Hexo 會依據在 _config.yml 中的設定，到 themes 資料夾中找尋相對應的圖片、CSS 與 JavaScript 檔案，以下面 _config.yml 裡的設定，Hexo 在生成靜態檔案的時候就會去 themes 資料夾找名字叫做 landscape 的這個主題。

所以我們只要把下載好的主題樣式檔案放在 themes 資料夾中，並修改 _config.yml 的設定就能更換主題。Hexo 官方網站上有許多現成的主體可以使用，假設你想修改 landscape 主題的 CSS檔，只要在 themes/landscape/source/css 找到對應的CSS檔就能直接修改。

若你想更動主題用到的圖片，則必須到 themes/landscape/source 中找到 _images 的資料夾，這邊的圖片未來將被用在產生靜態網站。

客製化自己的Url

  1. CNAME設定
    登入你的網域設定後台（GoDaddy、PChome 等等），在你的網域按新增 Record 並選擇 CNAME ，假設你買的網域名稱叫「hiPeter.com」，在 Host 欄位裡填 blog，你的部落格的網址就會是 blog.hiPeter.com ，接下來在「指向（Points）」的欄位填 username.github.io，記得把 username 換成你的 Github 帳號。

  1. 在 Github 打開你的 username.github.io，找到「Settings」頁面，裡面有個「Custom domain」，在裡面填入 http://blog.hiPeter.com

  2. 經過以上兩個步驟基本上就完成了，第一次連線可能會需要等幾分鐘到幾小時不等，不過當你重新 generate 並 deploy 新文章時，Github 會把之前設定好的「Custom domain」取消掉，這是因為設定網址的CNAME檔會被新 Deploy 檔案覆蓋掉，接下來我們要透過以下這個套件幫我們解決這個問題。

記得在 _config.yml 中的設定 Url。

之後每次 generate時就會重新生成CNAME的設定檔，如此一來就能解決這個問題囉！

Sitemap設定

  1. Sitemap
    npm install hexo-generator-sitemap –save
    之後重新啟動 hexo server，連到 /sitemap.xml 就能看到 Sitemap，

  2. 向 Google Search Console 提交Sitemap
    向 Google Search Console 提交 Sitemap 即可優化搜尋結果，此時 Google 會要求你驗證網站所有權，到 theme/landscape/layout/_partial/head.ejs 這個檔案，就能找到網站的 html meta 資訊，可在這邊插入 Google 的驗證程式碼。
