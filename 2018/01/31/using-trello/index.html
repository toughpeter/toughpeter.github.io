我使用trello的方式

面對工作生活中複雜的工作與頹團隊合作，透過一些工具能夠協助我們更加聚焦。

我選擇專案管理工具的考量點主要有以下幾點：

  1. 團隊協作人員：我的專案人數為五人以下，人員功能也較為單純，當團隊人數越多、工作內容功能越不同時，通常就需要更多的權限管理與流程管理。
  2. 容易上手好學習 選擇工具的第一步，先了解自己的專案複雜度，功能不用多，選擇最適合自己專案的，導入時也可以降低團隊學習門檻。
  3. 與其他工具生態系搭配 團隊是否有使用其他工具，像是Google Drive、Slack、Dropbox、Github、GitLab等工具？工具間是否有外掛可以串連資訊？

考量以上種種因素，我選擇Trello作為管理工具，如果你需要規模更龐大的專案管理可以參考Asana。

我怎麼使用Trello？

當你打開Trello首先映入眼簾的就是Trello的首頁，可以按照自己的需求創建團隊Team，並按照專案性質設計看板Board，如此一來團隊與看板就能一目了然。

  1. 最近常用的放到Starred Board
  2. 私人用的放一群
  3. 與團隊共用的，一個團隊開一群

點入Board後可以看到許多卡片Card放置於列表中，根據專案性質與團隊工作模式我們可以重新設計這些列表List與Card，舉例來說，產品團隊工作流程是：

  1. 開會討論要做的功能後，把大功能列出。
  2. 把接下來要做的功能拆解成實際要執行的任務，這個階段的重點是要把任務盡可能拆分到兩三小時內可以完成一張卡片的內容，降低時程錯估的機會，執行時若遇到困難需要更動時程時比較好重新評估。
  3. 決定要先執行的任務，由於專案執行時間有限，這個階段的重點是規劃出下次驗收時能夠使用的版本
  4. 驗收後的結果可能會需要修正或是增加新的功能。
  5. 有些資訊需要隨時能夠查詢到。
  6. 忽然蹦出的想法要能被記錄下來。

根據以上團隊的協作特性，我將Board中的List重新命名為以下幾欄，並賦予每欄有自己的權限與意義：

  • ISSUE，放置所有未解決的問題、功能：開完會拆解完功能成為小任務後由管理員將這邊的Card移到TODO中，有可能一張Card會被拆成很多張新的Card。
  • TODO，經過拆解後的問題，成為待辦任務：當執行者開始執行時可以將這邊的Card移到DOING中。
  • DOING，正在被執行的任務：當執行者執行完畢時可以將這邊的Card移到VERIFY中，讓管理者知道有事項需要驗收。
  • VERIFY，完成的任務，等這被驗收：管理者開始驗收時可以將這邊的Card移動到TESTING。
  • TESTING，正在驗收的任務：管理者驗收完沒問題後可以將這邊的Card移動到DONE，若驗收不過可以放回ISSUE或是新增新的問題到ISSUE中。
  • DONE，完成的任務，未來有需要可以回來察看當初討論的筆記。 INFO，存放永久的資料、事實、可能會用到的資訊，團隊成員可以隨時來這邊查詢，例如通訊錄、重要文件的連結。
  • IDEA，任何忽然想到的想法，不一定可以執行，未來可以討論。

舉一個工作情境作為說明：

  1. 想像今天要做一個功能，我們要在產品上加上一個會發光的燈泡，當產品開啟時燈泡會發光。 在ISSUE中新增一Card，「在產品上加上一個會發光的燈泡」。
  2. 開完時管理者與開發人員討論，將這個Card拆成新的五張Card，分別為「設計電路」、「更改外殼」、「鎖上燈泡」、「焊接電路」、「鎖上燈泡開關」，並放到TODO中，設定下次驗收時間為一週後。
  3. 隔天開發人員開始執行「設計電路」時就把「設計電路」從TODO移到DOING中，此時管理者也能查看，隨時了解進度。
  4. 因為這五張Card需要全部完成管理者才能測試，測試時管理者將Card移動到TESTING，完成後全部都移動到「DONE」，但是希望能透過開關改變燈泡的顏色，於是新增一張Card到「ISSUE」中
  5. 下次開會後重複以上流程

工具整合

團隊溝通主要是使用Slack作為溝通工具，可以透過Trello的外掛，當Trello有更新時或是標注某人時，也可以將訊息同步發送到Slack上。 Trello與Slack

你可能會運用其他工具像是Google Drive、Dropbox，或是你需要timeline、甘特圖的管理，這時候可以參考更強大的Trello功能，自己安裝自己需要的Add-ons、Extensions、Connectors、Power-Ups。 客製化你的Trello

結論

團隊溝通，建立適合團隊的流程，沒有絕對的方式。 設計自己的Trello符合你的工作流程，與團隊成員討論什麼樣的流程適合團隊，建立有利團隊成長的環境是成就一流團隊的第一步。找出適合自己的組合方式，工具是死的，有你活用才能發會工具的價值。
